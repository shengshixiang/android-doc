# 概要

通过带的项目,了解到pax一些比较重视的事项,后续新项目,可以项目开始阶段就注意重视起来.

让测试部,特别是硬件测试部,早点做测试

新项目立项先要看一下

# 新项目开始就要注意评估以下事项

* 关机续航,0电量关机还需要保持多少毫安的电量

因为要支持半年存储,不能ped触发,所以纽扣电池对sp的续航尤为重要

特别注意电池有没有对sp那边有电源支持,有的话,要注意0点量关机后,电池还有多少毫安的电量保存

* 温升,评估硬件布局,发热器件不应该搞在一起

特别注意高温,低温,温升是否符合标准. 

因为涉及到安全性,高温电池会不会鼓包等问题

器件主要分散安排,特别是charger,是否三防,屏蔽盖等

* 扫码,评估扫码时间,成功率等

因为涉及到支付,扫码成功率,速度等都比较关键.

特别是 运动容差等要符合标准

* 摄像头,效果定位等

摄像头看项目定位,一些高端的项目,摄像头用得比较好.

这个要早点确认模组,golden,otp等

调试很花时间,

* low mem device,包括评估是否编译go版本,就算非go版本,也要评估是否定义成低内存设备

软件是否一开始就要定义成低内存版本,特别是高android版本的项目

4G内存都不一定够用
